Google Java 编程风格

2.1 源文件以其最顶层的类名命名, 扩展名为.java, 大小写敏感
2.2 文件编码为 utf-8
2.3.1 空白字符, 只允许出现换行符和空格
2.3.2 特殊字符使用转义序列, 不使用8进制或Unicode转义
2.3.3 非ASCII字符, 可使用Unicode转义, 建议加上注释, 也可以不使用

源文件包含(按顺序):
    1. 许可证或版权信息
	2. package 语句
	3. import 语句
	4. 顶级类(只有一个,通常用public修饰)
	
	以上每个部分用一个空行隔开
	
3.1 许可证和版权信息应位于文件最前面
3.2 package 语句不换行
3.3.1 import 语句不使用通配符
3.3.2 import 语句不换行
3.3.3 顺序和间距
    import 语句可分为以下几组,按照这个顺序,每组由一个空行分隔：
	1. 静态导入
	2. com.google imports
	3. 第三方
	4. java imports
	5. javax imports
	组内不空行,按字典排序
3.4.1 每个文件只有一个顶级类并且与文件同名
3.4.2 类成员应按照某种逻辑排序(请不要按照添加的顺序)
3.4.2.1 重载:永不分离,重载的方法应按顺序出现在一起,中间不要存在任何方法

4.1 大扩号
4.1.1 大扩号与if, else, for, do, while 语句一起使用, 即使只有一行语句也要使用
4.1.2 非空块:大扩号遵循Kernighan和Ritchie风格
    1. 左大括号前不换行
    2. 左大括号后换行
    3. 右大括号前换行
    4. 右大括号后如果是一个块的结束则换行,否则不换行
4.1.3 一个空的块可简写为 {} ,不用换行, 如果是一个多块语句的一部分不可简写

4.2 每当开始一个新的块,缩进4个空格	
4.3 每个语句后要换行
4.4 列限制,不得超过规定的字符数,例外:
    1. 不可能满足列限制的行(如URL, JSNI)
    2. package 和 import 语句
	3. 注释中可能用到的shell
4.5 自动换行, 一行代码过长时, 为避免超出列限制而被分为多行
4.5.1 从哪里断开: 自动换行更倾向于在更高语法级别处断开
    1. 在 非赋值运算符 前断开(如 + . & |)
	2. 在 赋值运算符 后断开(如 =)
	3. 方法名或构造函数名与左括号留在同一行
	4. 逗号(,)与前面的内容留在同一行
4.5.2 自动换行时缩进至少+4个空格,同级语法元素可以使用相同的缩进

4.6 空白
4.6.1 垂直空白,以下情况要使用一个空行:
    1. 类内连续成员之间:字段, 构造函数, 方法, 嵌套类, 静态初始化块, 实例初始化块
	    tip:两个连续字段之间的空行是可选的,字段之间的空行主要用来对字段进行逻辑分组
	2. 方法体内,语句的逻辑分组间使用空行	
    3. 类内的第一个成员前和最后一个成员后的空行是可选的
4.6.2 水平空白:
    1. 分隔任何保留字与其后的左括号 "("
    2. 分隔任何保留字与其前面的右大括号 "}"
	3. 在任何左大括号前,例外:
        1. @SomeAnnotation({a, b})(不使用空格)
        2. String[][] x = foo;
    4. 在任何二元或三元运算符的两侧和类运算符。
    5. 在 , : ; 及右括号 ")" 后
    6. 如果在一条语后做注释,双斜杠
